# 支持的 Markdown 语法

本文档说明 markdown-html-ppt 技能支持的 Markdown 语法及其在 PowerPoint 中的呈现方式。

## 标题

### 一级标题 `#`

**Markdown 语法：**
```markdown
# 这是一级标题
```

**转换结果：**
- 创建一张标题幻灯片
- 标题居中显示
- 字体大小：44pt
- 字体样式：加粗

### 二级标题 `##`

**Markdown 语法：**
```markdown
## 这是二级标题
```

**转换结果：**
- 创建一张内容幻灯片
- 标题位于顶部
- 字体大小：28pt
- 字体样式：加粗

### 三级标题 `###`

**Markdown 语法：**
```markdown
### 这是三级标题
```

**转换结果：**
- 创建一张内容幻灯片
- 标题位于顶部
- 字体大小：28pt
- 字体样式：加粗

## 列表

### 无序列表

**Markdown 语法：**
```markdown
- 列表项 1
- 列表项 2
- 列表项 3
```

或使用 `*` 或 `+`：
```markdown
* 列表项 1
+ 列表项 2
```

**转换结果：**
- 每个列表项成为幻灯片中的一个要点
- 字体大小：14pt
- 自动添加项目符号

### 有序列表

**Markdown 语法：**
```markdown
1. 第一项
2. 第二项
3. 第三项
```

**转换结果：**
- 每个列表项成为幻灯片中的一个要点
- 字体大小：14pt
- 保留数字编号

## 段落

**Markdown 语法：**
```markdown
这是一个普通段落。

这是另一个段落。
```

**转换结果：**
- 段落文本作为幻灯片内容
- 字体大小：14pt
- 保留段落间的空行

## 文本格式

### 加粗

**Markdown 语法：**
```markdown
**加粗文本**
```

**转换结果：**
- 目前不支持行内格式
- 建议：使用标题或单独的列表项

### 斜体

**Markdown 语法：**
```markdown
*斜体文本*
```

**转换结果：**
- 目前不支持行内格式
- 建议：使用标题或单独的列表项

## 分页规则

### 自动分页

每个标题（`#`, `##`, `###`）会创建一张新的幻灯片：

**示例：**
```markdown
# 演示标题

## 第一部分
- 要点 1
- 要点 2

## 第二部分
- 要点 3
- 要点 4
```

**结果：**
- 幻灯片 1：标题幻灯片（演示标题）
- 幻灯片 2：内容幻灯片（第一部分）
- 幻灯片 3：内容幻灯片（第二部分）

### 内容归属

标题下的所有内容（列表、段落）归入该标题的幻灯片：

**示例：**
```markdown
## 项目目标

项目旨在提升用户体验。

- 优化界面设计
- 提高响应速度
- 增强安全性
```

**结果：**
- 创建一张标题为"项目目标"的幻灯片
- 包含段落和三个列表项

## 最佳实践

### 1. 使用清晰的标题层级

```markdown
# 主标题

## 章节 1
内容...

## 章节 2
内容...

### 子章节 2.1
内容...
```

### 2. 控制每张幻灯片的内容量

**推荐：**
```markdown
## 核心要点

- 要点 1
- 要点 2
- 要点 3
- 要点 4
- 要点 5
```

**避免：**
```markdown
## 过多内容

- 要点 1
- 要点 2
- 要点 3
- 要点 4
- 要点 5
- 要点 6
- 要点 7
- 要点 8
- 要点 9
- 要点 10
```

### 3. 使用简洁的标题

**推荐：**
```markdown
## 市场分析
```

**避免：**
```markdown
## 关于当前市场环境的详细分析报告以及未来发展趋势预测
```

### 4. 优先使用列表

列表格式在幻灯片中更易读：

**推荐：**
```markdown
## 项目优势

- 技术领先
- 成本优化
- 用户体验好
```

**避免：**
```markdown
## 项目优势

项目具有技术领先、成本优化、用户体验好等优势。
```

## 不支持的语法

以下 Markdown 语法目前不支持：

- 图片 `![alt](url)`
- 链接 `[text](url)`
- 代码块 ` ```code``` `
- 表格
- 引用 `> quote`
- 水平线 `---`
- 任务列表 `- [ ] task`
- 行内代码 `` `code` ``
